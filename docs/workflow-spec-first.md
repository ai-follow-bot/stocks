When implementing any new system or feature, first write a SPEC document (specification) describing the design, architecture, and implementation plan. Get user approval on the spec before writing any code. This applies to all new development work.

**Why:** Prevents wasted effort, ensures alignment on design decisions, and serves as reference documentation for the system.

**How to apply:** Start with a `SPEC_*.md` file in the project root. Cover: purpose, architecture, data flow, component design, integration points, and implementation plan. Only begin coding after user confirms.
